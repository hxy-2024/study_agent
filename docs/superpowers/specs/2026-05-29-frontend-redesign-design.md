# Frontend Redesign Design

## Product Direction

This redesign keeps the product local-first and personal. The UI should feel like a focused study console: fresh teal, compact, calm, and dense enough to avoid unnecessary scrolling. The visual reference is closer to GitHub Home than a marketing dashboard.

## Global Shell

Every page uses a top navigation bar:

- Hamburger button on the far left opens a drawer.
- Product icon and `study_agent` name sit next to the hamburger.
- User avatar sits on the far right.
- Drawer navigation contains `Spaces`, `Library`, `Reviews`, `Progress`, and `Settings`.
- The old persistent left sidebar is removed.

Settings are local-only for now. The settings surface stores default base URL, API key, default chat model, and optional embedding model in browser storage first. It can later be wired to an API. First-run guidance is optional and non-blocking.

## Home Dashboard

The home page is a GitHub-style dashboard:

- Left column lists all study spaces, newest first.
- The list has a search field and a `New` button on the same header row.
- Space rows show name, goal, status, and target days.
- The center area focuses on the current/most recent space and quick continuation.
- The right rail has a square calendar/event panel for diary and important reminders.
- The screen should fit common laptop viewports without forcing the user to scroll for primary actions.

## Create Study Space

The create page becomes a guided workflow:

- Top-left back arrow returns to the previous page.
- Step 1 collects study-space name, learning topic, and default model.
- Step 2 uploads material and starts RAG. The embedding model is optional.
- Embedded/chunked content is hidden by default and viewable through a centered modal with a close button.
- A sticky right-side route outline updates after ingestion/route generation. It is editable and includes an `AI Render` affordance with red-blue gradient text.
- `生成章节学习详情` triggers detailed chapter generation.
- While generating, show `正在生成中，请稍等`.
- Generated chapter details appear in a larger centered modal with more vertical space than the mockup. The modal has a back/close button at the top-left, chapter list on the left, chapter content on the right, and a bottom-right confirm action that enters chapter study.

Backend generation should eventually prefer uploaded material chapter structure. If no visible chapter structure exists, LangGraph backend prompts split the material. System prompts must remain backend-only and not be rendered in the UI.

## Chapter Study

The chapter page becomes chat-first:

- Left chapter rail lists chapters and has a `<<` collapse button.
- Main area is a large AI chat canvas.
- A default session is assumed for the current chapter.
- The first assistant message introduces the tutor and chapter focus. In the final backend shape this is generated by a second-layer agent and saved as an AI message.
- Messages render markdown.
- The composer includes attachment, model selector, thinking strength selector, and send button marked `↑`.
- A floating top-right session panel lists sessions, chapter progress, and notes.
- AI messages expose a checkpoint/fork button area.
- LangGraph interrupt, checkpoint, fork, web search, and MCP tools are represented in UI as reserved controls until all backend wiring is finished.

## Implementation Order

1. Global shell and home dashboard.
2. Create study-space workflow.
3. Chapter study chat-first page.

## Non-goals For This Slice

- No production login/account settings.
- No forced first-run setup.
- No full LangGraph checkpoint implementation in the browser if the backend route is not ready.
- No destructive migration or data reset.
